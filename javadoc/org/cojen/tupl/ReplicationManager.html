org.cojen.tupl

Interface ReplicationManager

  • All Superinterfaces:
    AutoCloseable, Closeable

    public interface ReplicationManager
extends Closeable
    Interface which replaces the redo log, for replicating transactional operations. Non-transactional operations, or those with NO_REDO durability will not pass through the replication manager.
    Author:
    Brian S O'Neill
    See Also:
    DatabaseConfig.replicate(org.cojen.tupl.ReplicationManager)

    • Method Detail

      • start

        void start(long position)
           throws IOException
        Start the replication manager in replica mode. Invocation of this method implies that all data lower than the given position is confirmed. All data at or higher than the given position might be discarded.

        After started, the reported position must match the one provided to this method. The position can change only after read and write operations have been performed.

        Parameters:
        position - position to start reading from; 0 is the lowest position
        Throws:
        IllegalArgumentException - if position is negative
        IllegalStateException - if already started
        IOException

        void recover(EventListener listener)

        Throws:

      • flip

        void flip()
        Called to acknowledge mode change from replica to leader, or vice versa. Until flip is called, all read and write operations fail as if the leadership mode is indeterminate. Reads fail as if the local instance is the leader, and writes fail as if the local instance is a replica.

      • notifyStore

        void notifyStore(Index index,
               byte[] key,
               byte[] value)
        Notification to replica when an entry is stored into an index. All notifications are uncommitted, and so loading with an appropriate lock mode is required for confirmation. The current thread is free to perform any blocking operations — it will not suspend replication processing unless all replication threads are consumed.
        Parameters:
        index - non-null index
        key - non-null key; contents must not be modified
        value - null if entry is deleted; contents can be modified
